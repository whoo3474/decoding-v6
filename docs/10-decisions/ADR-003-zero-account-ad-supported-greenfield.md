# ADR-003: 무계정·무료·광고 기반 그린필드와 수요 기반 데스크톱

- **날짜**: 2026-07-15
- **상태**: 승인
- **대체 범위**: ADR-002의 AI 설명·Pro/API 수익화·기존 코드 확장 전제
- **결정 문서**: `docs/prd/README.md` v6, `docs/prd/04-monetization.md`, `docs/prd/10-desktop.md`

## 맥락

범용 개발자 디코더라는 웨지는 유지할 가치가 있지만, 계정·해외 결제·구독·세금·고객 지원을 동시에 운영하면 1인 프로젝트의 복잡도가 제품 검증보다 먼저 커진다. 입력이 민감할 수 있는 도구에서 서버 AI와 사용자 계정은 “로컬에서 안전하게 판별한다”는 핵심 약속도 약화한다.

DevUtils 조사에서는 도구 수 자체보다 global hotkey, clipboard smart detection, 전용 inspector, offline 실행이 반복 사용의 핵심이라는 점을 확인했다. 다운로드 앱은 이 장점을 흡수할 수 있지만 웹 MVP와 동시에 만들면 플랫폼 signing·권한·업데이트 운영이 검증 전 범위를 과도하게 넓힌다.

## 결정

1. 기존 v3~v5 구현을 확장하지 않고 새 repository/worktree에서 v6를 그린필드로 구축한다.
2. 핵심 제품은 무료이며 로그인·계정·결제·구독·유료 API·생성형 AI를 만들지 않는다.
3. 디코딩·lint·설명은 브라우저/로컬 engine의 결정적 규칙과 공개 스펙으로만 수행한다.
4. 수익화는 트래픽·완료·성능 게이트 후 웹의 자체 호스팅 정적 스폰서 한 자리부터 시험한다. 개인화 광고, 리타기팅, payload 기반 광고는 금지한다.
5. PWA·CLI는 웹 engine을 재사용해 Phase 2에 제공한다.
6. 데스크톱은 웹 반복 수요가 증명된 뒤 Tauri 2로 제공한다. macOS universal을 먼저 안정화하고 Windows, Linux 순서로 확장한다.
7. 데스크톱은 무료·무로그인·무광고·기본 무텔레메트리다. shortcut을 누른 순간에만 clipboard를 읽고, signed update 외 네트워크는 없다.
8. DevUtils의 공개 47개 도구 전체를 Pack 1~4 확장 지도에 배치한다. 공통 operation registry·전용 inspector·local lazy chunk로 구현하고 arbitrary scripting·상시 clipboard 감시는 제외한다. 도구는 수요·성능·보안 게이트를 통과한 작은 slice로 추가한다.

## 수용하는 트레이드오프

- 구독 매출의 빠른 상한은 포기하지만 결제·환불·고객 계정·국가별 판매세 운영을 제거한다.
- 광고는 트래픽 전에는 수익이 없고 트래픽 후에도 작을 수 있다. 대신 제품의 무료·익명 접근성을 유지한다.
- 데스크톱 사용은 웹 광고 pageview를 줄일 수 있다. 이를 수익 손실보다 신뢰·반복 사용을 위한 배포 투자로 본다.
- Tauri는 플랫폼 webview 차이와 signing/notarization 운영을 만든다. 수요 게이트와 순차 출시로 범위를 제한한다.
- 기존 구현을 버려 초기 작성량이 늘지만, auth/payment/AI/legacy 복잡도가 새 제품으로 유입되지 않는다.

## 되돌리거나 재검토할 조건

- 공개 후 두 번의 런칭과 8주 관찰에서도 주간 완료 작업 300 미만 또는 4주 재방문 10% 미만이면 기능 확대 전에 웨지와 유통을 재검토한다.
- 광고가 운영비를 상회하지 않거나 활성·리텐션·Core Web Vitals 가드레일을 해치면 광고를 제거한다. 이를 이유로 자동으로 유료화하지 않는다.
- 데스크톱 수요 게이트를 충족하지 못하면 PWA·CLI에 집중한다.
- 계정·결제·AI·desktop 유료화로 전환하려면 별도 ADR만으로 부족하며 명시적인 제품 결정이 필요하다.
