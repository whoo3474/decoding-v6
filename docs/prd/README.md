## 1. 제품 정의

> **Paste anything. See what it is — instantly, locally, safely.**

정체불명의 문자열·바이너리(JWT, Base64, Hex, timestamp, certificate 등)를 붙여넣거나 파일로 놓으면 브라우저가 포맷을 자동 판별하고, 중첩된 인코딩을 재귀적으로 풀어 구조·위험·다음 행동을 보여준다.

### 절대 계약

1. **디코딩 입력과 결과는 사용자 기기를 떠나지 않는다.** 웹은 브라우저, CLI·desktop은 로컬 프로세스에서만 처리하며 광고·계측·URL·오류 로그에도 포함하지 않는다.
2. **로그인·계정·구독·결제·서버 동기화가 없다.** 모든 핵심 기능은 무료이며 익명으로 동작한다.
3. **서버 디코딩과 생성형 AI가 없다.** 설명은 결정적 규칙, 포맷 스펙, 계산된 근거로 만든다.
4. **광고는 제품보다 늦다.** 트래픽·성능 게이트 통과 후 비추적 문맥 광고 또는 자체 호스팅 스폰서 한 자리만 허용한다.
5. **새 코드베이스로 시작한다.** v3~v5 구현을 이동하거나 import하지 않고 필요한 자산만 감사 후 다시 작성한다.

## 2. 누구의 어떤 순간을 해결하는가

### 우선 사용자

- 인증·API 문제를 디버깅하는 애플리케이션 개발자
- 로그·페이로드·인증서를 조사하는 SRE/DevOps
- 출처가 불명확한 인코딩 데이터를 빠르게 분류하는 보안 엔지니어

### 핵심 Job-to-be-Done

> “이 값이 무엇인지 모르겠고 외부 서버에는 보내기 싫다. 10초 안에 정체와 다음 확인 지점을 알고 싶다.”

### 차별점

CyberChef의 연산 폭이나 단일 포맷 사이트의 검색 순위를 복제하지 않는다. decod.ing의 차별점은 다음 네 가지의 결합이다.

- 도구 선택 없이 붙여넣는 3초 UX
- 신뢰도와 애매함을 숨기지 않는 자동 판별
- 중첩 포맷을 한눈에 보여주는 안전한 체인 트리
- 로그인·업로드 없이 동작하는 감사 가능한 로컬 엔진

## 4. 북극성 지표와 KPI

**북극성**: 주간 **완료된 디코딩 작업 수**

단순 결과 렌더는 완료로 세지 않는다. 다음 중 하나가 발생해야 완료다.

- 사용자가 1위 후보를 유지한 채 결과를 복사·내보내기
- 다른 후보로 전환한 뒤 결과를 복사·내보내기
- 명시적 “Correct” 피드백

계측에는 페이로드, 결과, 해시, 정확한 길이, 로컬 사용자 ID를 포함하지 않는다. 이벤트 스키마는 PRD-09를 따른다.

| 축 | 핵심 지표 | 첫 목표 |
|---|---|---:|
| 획득 | 주간 비런칭 방문 | 1,000 |
| 활성 | 방문 → 완료된 디코딩 전환 | 50% 이상 |
| 품질 | 블라인드셋 top-1 precision / recall | 95% / 90% 이상 |
| 교정 | 결과 후 후보 전환율 | 5% 이하 |
| 속도 | 붙여넣기 → 첫 결과 p75 | 300ms 이하 |
| 리텐션 | 4주 내 재방문 브라우저 비율 | 20% 이상 |
| 수익 | 광고 적용 후 page RPM·순광고수익 | 비용 상회, 품질 가드레일 유지 |

리텐션은 원시 사용자 추적이 아니라 브라우저에 저장된 회전형 first-party 익명 cohort 토큰으로만 측정하며, 사용자가 분석을 끄면 생성하지 않는다.

## 7. 하지 않을 것

- 계정·로그인·이메일 수집·뉴스레터
- 구독·사용자 결제·유료 기능·API 과금
- 서버사이드 디코딩·페이로드 업로드·생성형 AI 설명
- 개인화 광고·리타기팅·광고 쿠키·데이터 브로커
- 입력·결과 영역 안의 광고, 팝업·전면·자동재생 광고
- 원시 입력의 기본 저장 또는 URL 포함
- 400개 연산 레시피 빌더와 근거 없는 47개 도구 복제
- 웹 수요가 증명되기 전의 데스크톱 앱 동시 개발
- 트래픽 근거 없는 다국어·도구 페이지 양산
- 기존 v3~v5 코드의 이동·import·부분 재활성화

## 8. 성공과 재검토 조건

### 성공

- 공개 후 8주 안에 주간 비런칭 방문 1K와 완료 전환 50% 달성
- 블라인드셋 품질 게이트를 포맷별로 통과
- 반복 사용자의 주요 경로가 웹·PWA·CLI 중 하나로 확인
- 광고 도입 시 제품 지표와 Core Web Vitals가 가드레일 안에서 유지

### 재검토

두 번의 공개 런칭과 8주 관찰 후에도 주간 완료 작업 300 미만이거나 4주 재방문 10% 미만이면 디텍터 수를 늘리지 않고 웨지·사용자·유통을 다시 검토한다. 광고 수익이 낮다는 이유만으로 제품 기능을 유료화하지 않는다.
